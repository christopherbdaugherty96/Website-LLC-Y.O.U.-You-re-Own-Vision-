# Revision Policy

Status: operating policy

Clear revision rules protect both the client and Auralis Digital.

---

## Standard Recommendation

Each website project should state how many revision rounds are included before work begins.

Suggested default:

- Website Refresh: 1 round
- Basic Website: 2 rounds
- Standard Website: 2-3 rounds
- Custom Projects: defined in quote

A revision round means one grouped set of feedback submitted together.

---

## What Counts as a Revision

Examples:

- wording changes
n- image swaps
- color adjustments
- section reordering
- CTA wording updates
- service description edits
- layout tweaks within agreed scope

---

## What Is Not a Revision

These usually require a new quote, added fee, or timeline change:

- full redesign direction change
- adding new pages not in scope
- adding booking/e-commerce/features later
- replacing all copy after approval
- major brand change mid-project
- custom integrations

---

## Feedback Rules

Recommended process:

- Client reviews draft
- Client sends one organized feedback list
- Auralis completes revision round
- Client reviews updated version

Scattered daily messages can slow projects and create confusion.

---

## Approval Point

Once final approval is given and the site is launched, new work may be billed separately unless included in a care plan.

---

## Internal Rule

Be flexible and reasonable, but do not allow unclear endless revisions to replace a defined scope.
