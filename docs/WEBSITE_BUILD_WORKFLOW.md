# Website Build Workflow

Status: operating workflow / production standard

This document defines the practical website-only workflow for Auralis Digital.

The goal is to lock down website creation first: consistent structure, strong visuals, mobile polish, and repeatable delivery.

---

## Core Rule

Do not start every website from scratch.

Auralis Digital should use a repeatable production system:

- same intake flow
- reusable layouts
- consistent design rules
- clean mobile process
- final quality checklist

Creativity should happen inside a reliable system, not replace the system.

## Standard Website Structure

Most client sites should start with this structure:

1. Hero section
2. Services / value section
3. Trust / proof section
4. Gallery / work section
5. Call-to-action section
6. Footer

Additional sections can be added when needed:

- about
- testimonials
- FAQ
- pricing
- booking
- map / service area
- before-and-after gallery
- blog / updates

## Design System Rules

Every site should have a simple design system before build polish begins.

Define:

- two fonts maximum
- one primary color
- one accent color
- background color rules
- button style
- card style
- section spacing
- image treatment
- mobile text sizing

Professional websites usually look good because they are consistent, not because every section is different.

---

## Layered / Cinematic Style Rules

Use cinematic styling carefully.

Good effects:

- layered hero sections
- dark overlays on strong photos
- subtle gradients
- soft shadows
- reveal animations
- hover lift effects
- image masks
- parallax used lightly
- staggered section reveals

Avoid:

- too much motion
- hard-to-read text over images
- random animations
- too many colors
- inconsistent spacing
- slow-loading effects

Motion should support the site, not become the site.

## Asset Strategy

Use the best available asset source for the client.

### Best Option: Real Client Photos

Use when the client has good photos or when Auralis can take photos.

Best for:

- storefront
- team
- services
- products
- workspace
- before-and-after results
- food / drinks / atmosphere

### Good Support Option: AI-Generated or Designed Assets

Use when real photos are missing or when decorative visuals are needed.

Best for:

- abstract backgrounds
- icons
- hero mood visuals
- section art
- brand textures
- placeholder concepts

### Rule

Do not use AI visuals to fake real staff, fake locations, fake testimonials, fake results, or fake client work.

Authenticity matters for local business trust.

## Client Intake

Before building, collect:

- business name
- business type
- services
- location / service area
- phone
- email
- booking link
- social links
- photos / logo
- testimonials
- preferred style
- disliked styles
- competitors or references
- main goal: calls, leads, bookings, orders, visits

No good intake means no good website.

## Production Workflow

### Step 1 — Intake Review

Understand the business, goal, audience, and available assets.

### Step 2 — Pick Template Type

Choose:

- Template A: Local Service
- Template B: Premium Brand
- Template C: Visual / Hospitality

### Step 3 — Wireframe First

Plan the page in simple structure before styling.

Decide:

- section order
- headline placement
- CTA placement
- image placement
- trust sections

### Step 4 — Write Copy

Write clear, conversion-focused copy:

- headline
- subheadline
- service descriptions
- trust points
- CTA text
- FAQ answers

### Step 5 — Build Visual System

Apply:

- fonts
- colors
- spacing
- buttons
- cards
- image treatments

### Step 6 — Add Photos / Assets

Use client photos first.
Use AI/design assets only where helpful and honest.

### Step 7 — Add Motion Last

Only after layout, copy, and visuals are strong.

Motion examples:

- fade up
- hover lift
- subtle parallax
- staggered reveals

### Step 8 — Mobile Polish

Every website must be checked on mobile.

Review:

- text size
- spacing
- nav
- buttons
- image crops
- forms
- CTA visibility

### Step 9 — Final QA

Check:

- links
- buttons
- forms
- spelling
- responsiveness
- image loading
- contact info
- SEO basics
- page speed basics

### Step 10 — Client Review

Ask focused review questions:

1. What do you like most?
2. What feels off?
3. What action should customers take first?

Avoid vague endless revision cycles.

## 30-Day Website Lockdown Plan

### Week 1 — Layout Practice

Recreate three strong homepage layouts.

Goal:
learn section structure, spacing, and hierarchy.

### Week 2 — Build Base Templates

Create three reusable templates:

- local service business
- premium brand
- visual / hospitality business

### Week 3 — Build Fake Client Projects

Create three sample sites using fake or practice businesses.

Goal:
build speed and confidence.

### Week 4 — Portfolio and Selling

Turn the best examples into portfolio pieces.
Start offering websites to real businesses.

---

## Quality Checklist

Before delivery, every site should answer yes to these:

- Is the main offer clear in five seconds?
- Is the CTA obvious?
- Does it look good on mobile?
- Are services easy to understand?
- Are contact options easy to find?
- Are images high quality?
- Is spacing consistent?
- Are fonts readable?
- Does motion feel subtle and professional?
- Does the site feel trustworthy?

---

## What To Avoid

Avoid:

- starting every site from scratch
- overusing animations
- using too many colors
- ignoring mobile
- weak spacing
- weak CTAs
- generic AI-looking visuals
- fake-looking brand claims
- endless tool switching
- polishing forever before selling

---

## Final Standard

Auralis Digital does not need to know everything to deliver good websites.

The standard is:

**good structure + good visuals + mobile polish + fast delivery + honest client communication.**
